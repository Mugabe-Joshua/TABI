How does code move using Git

Code lives in 4 areas:
1. The working directory 2. The Index 3. The Local repo 4. The Remote repo

The working directory:

This is where you make changes directly to local code.
Imagine it as the creation point.

The Index:

This is where changes are added for the next commit.
Imagine it as the bridge between the working directory and the local repo.

The local repo:

This is where commits of code are tracked by Git for security purposes. However this is done locally.
Imagine this as the introvert twin.

The remote repo:

This is where code can be pushed for collaborating with other developers and are tracked by Git. This is done online.
Imagine this as the extrovert twin.
NB:
When pushing local changes to a remote repo for the first time, we use the command (git push --set-upstream origin master) so as to inform the introvert twin that it can share its stories with its extrovert twin only.

CLONING A REMOTE REPO

Use the command(git clone "repo_URL") We can decide to create a branch other than the default master branch using the git command(git checkout -b "new_branch_name")
